前端交互模式

用户如何与三个 Agent 交互？

A

统一对话窗口

所有 Agent 共用同一个聊天界面，靠意图识别自动路由

帮我出5道第三章的题
📝 出题 Agent：已生成5道选择题...
导数的几何意义是什么
💬 问答 Agent：导数几何意义是...

Pros

  • 体验统一，不用学
  • 实现简单

Cons

  • 出题结果嵌在对话里，不好单独查看
  • 诊断报告在聊天流中不够突出
B

对话 + 功能面板（推荐）

左侧统一对话，右侧根据当前 Agent 展示对应的功能面板

出5道第三章的题
好的，已生成↓
📝 出题面板
Q1. 导数的定义是？
A. ... B. ...
Q2. ...

Pros

  • 出题有独立答题区
  • 诊断报告可视化展示
  • 对话历史和功能面板各司其职

Cons

  • UI 更复杂
  • 右侧面板需要根据上下文动态切换
C

三 Tab 分离

顶部三个 Tab（问答/出题/诊断），每个 Tab 内是独立对话

💬 问答 | 📝 出题 | 📊 诊断
当前 Tab: 出题
出题 Agent 专属对话区

Pros

  • 功能边界清晰
  • 各自上下文独立

Cons

  • 用户需要手动切换 Tab
  • 跨 Tab 信息不互通
  • 和混合模式调度器冲突
