调度器工作模式 — 你选哪种？

调度器决定用户请求如何路由到对应的 Agent

A

自动意图识别

用户随便说，调度器用 LLM 自动判断意图 → 路由到对应 Agent

例："导数是啥" → 问答Agent | "出5道题" → 出题Agent | "我哪些不会" → 诊断Agent

Pros

  • 用户体验最自然
  • 不需要学命令

Cons

  • 意图判断可能出错
  • 边界模糊时难处理
B

显式模式切换

用户主动选择当前模式：问答模式 / 出题模式 / 诊断模式

例：点击"出题模式"按钮 → 之后所有对话都按出题处理

Pros

  • 不会误判
  • 用户明确知道在做什么

Cons

  • 多一步操作
  • 用户可能忘记切换
C

混合模式（推荐）

默认自动意图识别，但也提供手动切换按钮作为兜底

自动识别置信度高 → 直接路由 | 置信度低 → 弹出选项让用户确认

Pros

  • 兼顾流畅性和准确性
  • 低置信度时不会做错事

Cons

  • 实现稍复杂
  • 需要定义置信度阈值
