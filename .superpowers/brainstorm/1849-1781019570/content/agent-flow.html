多意图场景：串行 vs 并行

当用户一句话包含多个意图时，Agent 怎么协作？

示例场景

"帮我出5道第三章的题，顺便分析下我哪些地方薄弱"

→ 包含两个意图：出题 + 诊断

A

串行 — 排队处理

调度器拆成两个任务，排队执行，用户逐步看到结果

1️⃣ 出题 Agent 先生成5道题
2️⃣ 诊断 Agent 再分析薄弱点
3️⃣ 汇总展示

Pros

  • 实现简单
  • 结果不会互相干扰

Cons

  • 用户等待时间长
  • 两个结果之间没有联动
B

并行 — 同时执行（推荐）

出题和诊断同时跑，结果汇总后一起展示

⚡ 出题 Agent + 诊断 Agent 同时启动
⚡ 各自独立执行
⚡ 汇总后统一返回

Pros

  • 响应快（只等最慢的那个）
  • 可以联动（诊断结果影响出题难度）

Cons

  • LLM 并发调用成本高
  • 结果汇总逻辑复杂
C

智能选择

简单意图串行，独立意图并行，依赖意图先跑前置再跑后置

🧠 调度器判断：
• 独立意图（出题+诊断）→ 并行
• 依赖意图（出题→根据错题诊断）→ 串行
• 单意图 → 直接执行

Pros

  • 最优体验
  • 灵活适应各种场景

Cons

  • 实现最复杂
  • 依赖判断可能出错
