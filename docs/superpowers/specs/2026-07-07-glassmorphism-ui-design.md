# Glassmorphism UI Design Spec

## Goal

将当前 AI 学习工作台前端重塑为“清透彩色玻璃拟态”的产品级界面。目标不是做展示型炫技页面，而是让现有问答、测验、复习工作流拥有更强的空间层次、AI 操作台气质和长期可读性。

## Final Prompt

请将当前 AI 学习工作台前端整体重塑为“清透彩色玻璃拟态”的产品级界面。

### 核心风格

使用低饱和极光彩云背景，整体保持浅色、清透、柔和。背景以雾白为底，叠加蓝、青、紫三色的低饱和渐变光晕，呈现清晨雾面玻璃上的柔和极光感。界面气质应结合“智能 AI 操作台”和“苹果式空间感”，不要做成深色赛博风、霓虹娱乐风或营销落地页。

### 玻璃系统

采用中等磨砂和分层玻璃。重点区域使用较强玻璃质感：侧边栏、顶部栏、首页快捷卡片、聊天输入框、主操作按钮、模式切换、弹窗。正文阅读区域使用轻玻璃：聊天气泡、测验题目卡、错题详情、解释文本，优先保证文字可读性。

凡是承载长文本的区域必须有足够不透明度的阅读底板。正文面板背景透明度不低于 0.72。聊天 AI 气泡、题目解析、错题详情优先可读性，不允许文字直接压在高变化渐变背景上。

### 形状语言

采用苹果式大圆角和胶囊按钮重点。大容器、浮层、首页卡片、输入框使用更圆润的大圆角，营造空间浮层感。模式切换、主按钮、状态标签、快捷操作使用胶囊形态。错题列表、资料列表、普通表单控件保持更克制的中圆角，避免界面玩具化。

### 颜色规则

主高光使用蓝青紫三色系统：

- 蓝色用于主操作、当前模式、发送、上传、选中状态。
- 青色用于聚焦光晕、处理中状态、辅助高光。
- 紫色用于 AI 氛围光、背景过渡、少量强调。

颜色必须低饱和、浅色、透明叠加，不要使用大面积高饱和紫色。文本保持深墨蓝，次级文本保持可读的蓝灰色。

### 模式差异

问答模式最清透、最轻盈，突出聊天输入框、聊天气泡和引用标签的玻璃层次。

测验模式更聚焦，题目卡片和选项保持高可读性，选中、正确、错误状态用清晰边框、图标和轻微高光表达。

复习模式更像玻璃数据工作台，筛选、错题列表、详情面板、统计条采用更密集但清晰的分层玻璃布局。

全局侧边栏和顶部栏作为强玻璃导航层，在所有模式中保持统一。

### 信息密度

首页和问答模式可以更宽松、更有空间感。复习页保持较高信息密度，方便扫描、筛选和对比错题。不要为了玻璃效果牺牲工具效率。

### 动效

使用更强但高级的空间动效：

- 页面或模式切换时，玻璃浮层轻微位移和淡入。
- 首页卡片进入时可以轻微 stagger。
- 卡片 hover 微浮起并增强边缘高光。
- 输入框 focus 出现蓝青紫柔和光晕。
- 上传、生成、处理中使用克制的流动光效。

所有动画必须使用 transform 或 opacity，时长控制在 150-400ms，并支持 `prefers-reduced-motion` 降级。

## Design Tokens

所有玻璃效果必须通过语义 CSS token 管理。组件中避免散落 raw `rgba`、`blur`、`shadow`。

建议 token 类别：

- `--glass-bg-strong`
- `--glass-bg-medium`
- `--glass-bg-soft`
- `--glass-border`
- `--glass-border-strong`
- `--glass-blur-strong`
- `--glass-blur-medium`
- `--glass-blur-soft`
- `--glass-shadow-soft`
- `--glass-shadow-float`
- `--glow-blue`
- `--glow-cyan`
- `--glow-purple`
- `--radius-float`
- `--radius-panel`
- `--radius-control`
- `--radius-pill`
- `--motion-glass`
- `--z-background`
- `--z-content`
- `--z-navigation`
- `--z-input`
- `--z-scrim`
- `--z-dialog`
- `--z-toast`

## Component Application

### App Shell

使用柔和极光渐变背景，不使用离散装饰圆球。背景应像连续的低饱和彩云，而不是明显的 bokeh 或独立光斑。

### Sidebar

侧边栏是强玻璃导航层：半透明、明显 blur、白色细边框、柔和阴影。选中会话和选中文档使用蓝色竖线、胶囊高光或轻微内发光表达。

### Header

顶部栏是强玻璃悬浮层。模式切换使用胶囊 segmented control，当前模式用蓝青渐变或蓝色玻璃底表达。顶部栏需要保持 sticky/fixed 关系清晰，不能遮挡内容。

### Quick Actions

首页三张快捷卡使用大圆角玻璃浮层，并带蓝青紫边缘高光。首页可以比工具页更宽松、更有视觉氛围。

### Chat Input

底部输入框是核心玻璃浮层。默认状态清透，focus 状态出现柔和蓝青紫光晕。发送按钮使用胶囊或圆形蓝青渐变按钮，disabled 状态必须清楚。

### Message Bubbles

用户消息可用蓝色玻璃胶囊，AI 消息用白色轻玻璃。长文本必须保持高对比和稳定行高。引用标签使用小胶囊玻璃标签。

### Quiz Card

题目区轻玻璃，选项按钮保持清晰边界。正确、错误、选中状态不能只依赖颜色，必须保留图标、边框、背景或文案提示。

### Review Workspace

筛选面板、错题列表、详情面板统一玻璃数据面板。复习页信息密度比首页更紧凑，长列表项不应每一行都使用高强度 `backdrop-filter`。可以用半透明背景、边框、阴影模拟轻玻璃，以控制性能。

### Dialogs And Toasts

弹窗使用玻璃面板和足够强的 scrim，保证前景可读。scrim 不能太弱，避免背景和弹窗竞争。

### Buttons

主按钮使用蓝青渐变胶囊。次级按钮使用白色玻璃。危险按钮保持语义红色但降低饱和，并与普通操作拉开视觉距离。

## Engineering Constraints

### Performance

`backdrop-filter` 只用于导航、输入框、浮层、主要卡片等关键区域。避免对长列表每一项使用高强度 blur。移动端降低 blur 强度。不要创建大面积全屏高 blur 叠层。

### State System

每个交互组件必须覆盖：

- hover
- active
- focus-visible
- disabled
- selected
- loading
- error

状态表达不能只依赖颜色，必须结合边框、图标、透明度、文案或形态变化。

### Z-Index

定义清晰层级：

- 背景：0
- 内容：10
- 侧边栏和顶部栏：20
- 底部输入框：30
- 弹窗 scrim：50
- 弹窗面板：60
- toast 或 dev overlay：70

不要使用任意超大 z-index。

### Scope Boundary

本次改造以样式系统和表现层为主，不改变 API、store、业务流程、资料上传逻辑、测验评分逻辑和复习数据逻辑。除非为了添加必要 `className` 或 aria 属性，否则避免重写组件结构。

## Accessibility And Quality

文字对比度必须满足可读性，不能让透明背景降低正文可读性。所有可点击目标至少 44px 高。保持 lucide 图标风格一致。不要使用 emoji 作为结构图标。不要出现水平滚动、文本重叠、按钮文字溢出。移动端必须重新检查侧边栏、顶部栏、输入框和卡片布局。

## Verification

实现后需要运行：

- 相关文件格式检查。
- Next.js 生产构建。
- Header、AppSidebar、ChatInput、MessageList、QuizCard、ReviewWorkbench 等相关组件测试。
- 浏览器检查桌面和 390px 移动端截图。

重点确认：

- CSS 正常加载。
- 玻璃背景不过度干扰。
- 复习页信息仍可扫描。
- 输入框和按钮状态清晰。
- 侧边栏、顶部栏和底部输入框不互相遮挡。
- `prefers-reduced-motion` 下动效降级。
